# Taller Individual: Node.js

## Descripción

Este taller tiene como objetivo reforzar los conocimientos adquiridos sobre Node.js. Durante el desarrollo del taller, trabajarás con un proyecto que contiene algunos errores que deberás identificar y corregir. Además, implementarás validaciones para los campos de entrada y agregarás nuevas funcionalidades relacionadas con el módulo de libros.

## Objetivos del Taller

1. **Identificar y corregir errores** en el código existente.
2. **Agregar validaciones** para los campos de entrada.
3. **Implementar funcionalidades** para el módulo de libros:
    - Agregar libros.
    - Actualizar libros.

## Requisitos Previos

Antes de comenzar, asegúrate de tener instalado lo siguiente:

- [Node.js](https://nodejs.org/) (versión 20 o superior).
- Un editor de texto como [Visual Studio Code](https://code.visualstudio.com/).
- Conocimientos básicos de JavaScript y Node.js.

## Instrucciones

1. **Clona el repositorio** del taller en tu máquina local:
    ```bash
    git clone <URL_DEL_REPOSITORIO>
    cd taller_individual_node
    ```

2. **Instala las dependencias** del proyecto:
    ```bash
    npm install
    ```

3. **Revisa el código existente**:
    - Identifica los errores en el código y corrígelos.
    - Asegúrate de que el proyecto funcione correctamente después de realizar los cambios.

4. **Agrega validaciones**:
    - Implementa validaciones para los campos de entrada en las rutas existentes.
    - Asegúrate de manejar errores de manera adecuada y devolver respuestas claras al cliente.

5. **Implementa las nuevas funcionalidades**:
    - **Agregar libros**: Crea una ruta para agregar nuevos libros al sistema.
    - **Actualizar libros**: Crea una ruta para actualizar la información de libros existentes.
    - **Buscar libros por autor**: Crea una ruta para actualizar la información de libros existentes.
    

6. **Prueba tu solución**:
    - Verifica que todas las funcionalidades trabajen correctamente.
    - Asegúrate de que las validaciones y los mensajes de error sean claros.

## Entregables

- Código corregido y funcional.
- Validaciones implementadas.
- Funcionalidades de agregar y actualizar libros completadas.
- Documentación breve de las rutas y cómo probarlas.

## Evaluación

Se evaluará lo siguiente:

1. Corrección de errores en el código.
2. Implementación de validaciones.
3. Correcta implementación de las funcionalidades solicitadas.
4. Calidad del código (legibilidad, buenas prácticas, etc.).
